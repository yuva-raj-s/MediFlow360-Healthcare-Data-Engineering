02

Infrastructure as Code (IaC)

Industry standard projects don't click buttons in the Azure Portal. We use Terraform or Bicep to provision resources. This ensures consistency across Dev, UAT, and Prod environments.

[TF] terraform init
[TF] terraform apply -var="env=prod"
[TF] + azurerm_storage_account.adls (CREATED)
03

Bronze: The Raw Landing Zone

The Bronze layer is Immutable. We ingest data exactly as it appears in the source systems. We add Metadata columns like _load_timestamp and _run_id for full lineage tracking.

// BEGINNER TIP: Never transform data in Bronze. Keep it raw to allow for easy reprocessing if logic changes.

04

Silver: The Cleanse & Masking

The Silver layer is where the heavy lifting happens. We apply SCD Type 2 for historical tracking and use our PII Masking Engine to ensure HIPAA compliance.

# Silver Logic Example df_silver = df_bronze.withColumn("masked_email", mask_email_func(col("email"))) df_silver.write.format("delta").saveAsTable("silver.patients")
05

Gold: Business Insights

Gold tables are designed for the business. They are usually Aggregated or Denormalized for high-performance reporting in Power BI.

Example: kpi_daily_summary joins Patients, Lab Results, and Claims to calculate Cost-per-Patient per day.

06

Governance & PII

Healthcare data is sensitive. We implement Unity Catalog to manage permissions. Only authorized engineers see Bronze; Analysts only see Gold.

Critical Security Rule

Never store Aadhaar or Phone numbers in plain text. Use the PII Masking Engine at the earliest possible stage.

07

Deployment & CI/CD

We use Azure DevOps Pipelines to automate everything. Every code push triggers a "Secret Scan" to ensure no passwords or PII leaked into the codebase.
